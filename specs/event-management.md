# イベント管理

## 機能概要

特別なイベント（イースター、クリスマス礼拝など）がある日曜日を「イベント日」として設定し、その日はヘルパー（HELPER）をリーダー割り当ての対象外とする機能。

## ユースケース

### 正常系

1. **イベント日の設定**: スケジュール画面で特定の日曜日を「イベント日」に設定できる
2. **イベント日の解除**: 設定済みのイベント日を通常日に戻せる
3. **イベント日の割り当て生成**: イベント日に自動生成を行うと、HELPERメンバーが候補から除外される
4. **イベント日の手動調整**: イベント日の差し替え候補にもHELPERが表示されない

### 異常系

1. **除外日かつイベント日**: 除外日（isExcluded）に設定されている日はそもそも割り当て対象外のため、イベントフラグは無関係（両方設定可能だが、除外日が優先）

## 入出力の定義

### API

#### イベント日の切り替え

```
POST /api/schedules/:id/toggle-event
Response: { id, date, isExcluded, isEvent, year }
```

- トグル動作: 現在の `isEvent` を反転させる
- レスポンスは更新後のスケジュール情報

### データベース

#### マイグレーション（004-schedule-events）

```sql
ALTER TABLE schedules ADD COLUMN is_event INTEGER NOT NULL DEFAULT 0;
```

## ドメインモデル

### Schedule エンティティの拡張

- `isEvent: boolean` プロパティを追加
- `toggleEvent(): Schedule` メソッドを追加（`toggleExclusion()` と同様のイミュータブルパターン）
- `create()` / `reconstruct()` に `isEvent` を含める

### assignment-generator の変更

- `selectPair` のメンバーフィルタリングで、スケジュールが `isEvent` の場合に `MemberType.HELPER` のメンバーを除外する

### constraint-checker の変更

- 変更なし（イベント日は制約違反ではなく、候補フィルタリングで対応）

## 制約・ビジネスルール

1. イベント日かつ除外日の場合、除外日が優先される（割り当て自体が生成されない）
2. イベント日のリーダー割り当て候補から `MemberType.HELPER` のメンバーは完全に除外される
3. イベント日であっても `PARENT_COUPLE` / `PARENT_SINGLE` のメンバーは通常通り割り当て対象となる
4. 手動調整（差し替え）時の候補一覧でも、イベント日の場合はHELPERを除外する

## UI

### スケジュール画面

- 各日曜日カードに「イベント」トグルボタンを追加
- イベント日はカードに視覚的な区別を付ける（アイコンまたはラベル表示）
- 除外ボタンとイベントボタンは独立して操作可能

### 割り当て画面

- イベント日の割り当てグループにイベントラベルを表示
- 差し替え候補のドロップダウンからHELPERが除外されていること

## 受け入れ基準（テスト観点）

### ドメイン層

- [ ] `Schedule.create()` で `isEvent` がデフォルト `false` であること
- [ ] `Schedule.toggleEvent()` で `isEvent` が反転すること
- [ ] `toggleEvent()` が新しい Schedule インスタンスを返すこと（イミュータブル）

### ドメインサービス層

- [ ] `isEvent` なスケジュールに対して `assignForSchedule` を実行すると、HELPER メンバーが割り当てられないこと
- [ ] `isEvent` でないスケジュールでは HELPER メンバーが通常通り割り当て対象になること
- [ ] HELPER のみで構成されたメンバーリストでイベント日に生成した場合、割り当て可能なペアがないこと

### アプリケーション層

- [ ] 差し替え候補 API がイベント日の場合に HELPER を除外すること

### インフラ層

- [ ] マイグレーションで `is_event` カラムが追加されること
- [ ] `SqliteScheduleRepository` で `isEvent` の読み書きが正しく動作すること

### プレゼンテーション層

- [ ] `POST /api/schedules/:id/toggle-event` でイベントフラグがトグルされること
- [ ] スケジュール画面にイベントトグルボタンが表示されること

### i18n

- [ ] `event` / `eventDay` のキーが日英両方に追加されていること
