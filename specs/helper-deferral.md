# 仕様書: ヘルパー後回し（HELPER deferral）

## 機能概要

割り当て生成時、ヘルパー（HELPER）を含むペアにソフトペナルティを加算し、同等条件では親（PARENT_COUPLE / PARENT_SINGLE）を優先的に選出する。

### 背景

- ヘルパーは「助けてくれる方」であり、親が主体的に担うべき活動
- 現在のスコアリングではメンバータイプによる優先度差がなく、同スコアなら等確率で選出される
- 親が十分いる場合でもヘルパーが選ばれることがあり、実態と合わない

## ビジネスルール

### R1: ヘルパーペナルティの追加

`scorePair()` でペア内のヘルパー1人につき **+5** を加算する。

```
HELPER 0人: +0
HELPER 1人: +5
HELPER 2人: +10
```

### R2: 既存制約との優先度関係

ヘルパーペナルティはソフト制約であり、以下の関係を持つ:

| 制約 | ペナルティ | ヘルパーペナルティとの関係 |
|------|-----------|--------------------------|
| 言語バランス | +100000 | ハード制約が常に優先 |
| 同性制約 | +100000 | ハード制約が常に優先 |
| CLASS_LANGUAGE_COVERAGE | +100000 | ハード制約が常に優先 |
| 月内重複 | +100/人 | 重複回避が優先 |
| 均等配分 | +50/回差 | count差1回分でヘルパーペナルティを上回る |
| 配偶者回避 | +30 | 配偶者回避が優先 |
| 日付指定優先 | -30/人 | 日付指定が優先 |
| ペア多様性 | +10/回 | 1回のペア重複でヘルパーペナルティと同等 |
| **ヘルパー後回し** | **+5/人** | — |
| BOTH温存（合同日） | +3/人 | ヘルパーペナルティが上回る |
| BOTH温存（分級日G1） | +3〜5 | 同程度 |

### R3: 均等配分との共存

- 親のcount > ヘルパーのcount の場合、均等配分ペナルティ（+50/回差）がヘルパーペナルティ（+5）を大きく上回るため、ヘルパーが選出される
- これは正しい動作: ヘルパーが少ない回数しか担当していなければ、公平性のために選出されるべき

### R4: イベント日の既存ルールは変更なし

イベント日のヘルパー除外（ハード制約）は従来通り。本仕様はスコアリングのソフト制約のみ。

## ユースケース

### UC1: 同スコアで親ペアとヘルパーペアが競合

**前提:** UPPER プールに JP親×2、EN親×1、ENヘルパー×1（全員 count=0）

**処理:**
- JP親+EN親ペア: score = 0
- JP親+ENヘルパーペア: score = +5

**結果:** JP親+EN親ペアが優先される。

### UC2: ヘルパーのcountが親より少ない場合

**前提:** JP親(count=2) + EN親(count=0) vs JP親(count=2) + ENヘルパー(count=0)

**処理:**
- JP親+EN親: 均等配分 (2-0)×50 = +100, ヘルパーペナルティ +0 → score = +100
- JP親+ENヘルパー: 均等配分 (2-0)×50 = +100, ヘルパーペナルティ +5 → score = +105

**結果:** まだ親ペアが優先されるが、差はわずか（+5）。

### UC3: ヘルパーのcountが十分低い場合

**前提:** JP親(count=3) + EN親(count=2) vs JP親(count=3) + ENヘルパー(count=0)

**処理:**
- JP親+EN親: 均等配分 (3-0)×50 + (2-0)×50 = +250 → score = +250
- JP親+ENヘルパー: 均等配分 (3-0)×50 + (0-0)×50 = +150, ヘルパーペナルティ +5 → score = +155

**結果:** ヘルパーペアが選出される（均等配分が勝つ）。正しい動作。
