# 仕様書: 自動組み合わせ生成アルゴリズム

## 機能概要
全制約を考慮し、各日曜日のリーダー割り当て（2グループ×2人＝計4人）を自動生成する核心機能。

## ドメインモデル

### エンティティ: Assignment
| 属性 | 型 | 説明 |
|------|-----|------|
| id | AssignmentId (UUID) | 一意識別子 |
| scheduleId | ScheduleId | 対象日のスケジュールID |
| groupNumber | 1 \| 2 | グループ番号 |
| memberIds | [MemberId, MemberId] | ペアのメンバーID（MVP: 2人固定） |

### 値オブジェクト
- **AssignmentResult**: 1日分の割り当て結果（Assignment × 2グループ）
- **ConstraintViolation**: 制約違反情報（違反種別、対象メンバー、重大度）

## 制約条件（優先度順: 高→低）

### 必須制約（緩めない）
1. **参加可能日の制約**: 参加不可の日にメンバーを割り当てない
2. **言語バランス**: 各グループに英語対応者と日本語対応者を最低1人ずつ配置

### 強い制約（緩めにくい）
3. **同性ペア制限**: sameGenderOnly=trueのメンバーは同性としか組まない（ペアが2人の場合のみ適用）

### 弱い制約（必要に応じて緩める）
4. **月内重複回避**: 同じ人が同月内に2回以上担当しない
5. **均等な担当回数**: 全メンバーの担当回数の差を最小化する
6. **夫婦回避**: 同じ日の同じグループにPARENT_COUPLEの夫婦を配置しない（PARENT_SINGLE・HELPERには適用しない）
7. **満遍なくペアを組む**: 過去に組んだことのないペアを優先する

### 均等性ルール
- 担当回数を均等にする
- 最低2週間は間隔を空ける
- 均等性の計算は年度（4月〜翌3月）単位でリセットする
- 途中参加メンバーは参加時点からカウント開始

## ユースケース

### UC-1: 自動割り当て生成
- **入力**: 対象年月
- **前提条件**: 対象月のスケジュールが存在する、有効メンバーが十分にいる
- **正常系**: 対象月の全有効日曜日の割り当てを生成する
- **制約違反時**: 優先度の低い制約から順に緩和し、緩和した制約を警告として返す
- **出力**: AssignmentResult[] と ConstraintViolation[]（警告）

### UC-2: 割り当ての手動調整
- **入力**: AssignmentId、変更するメンバーID
- **正常系**: 指定のペアメンバーを差し替える
- **注意**: 手動調整時にも制約チェックを行い、違反があれば警告を表示（確定は可能）

### UC-3: 割り当て削除
- **入力**: スケジュールIDまたは年月
- **正常系**: 対象の割り当てを削除する（再生成用）

### UC-4: 欠席対応（手動差し替え）
- **入力**: AssignmentId、欠席メンバーID、代理メンバーID
- **正常系**: ペアメンバーを差し替える
- **注意**: 欠席者の担当回数にはカウントしない

## アルゴリズム概要
1. 対象月の有効日曜日を取得
2. 有効メンバーを取得し、参加可能日でフィルタ
3. 各日曜日について:
   a. 高学年担当（UPPER）から2名、低学年担当（LOWER）から2名を選出
   b. 必須制約を満たす候補を抽出
   c. 強い制約→弱い制約の順にスコアリングし、最適な組み合わせを決定
   d. 制約を緩和した場合は ConstraintViolation を記録
4. 全日曜日の割り当てを返す

## グループ構成ルール（MVP）
- グループ1: 高学年担当1名 + 低学年担当1名
- グループ2: 高学年担当1名 + 低学年担当1名
- 各グループに英語・日本語の対応者を配置

## 受け入れ基準
- [ ] 1ヶ月分の割り当てが自動生成される
- [ ] 各日に4名（2グループ×2人）が割り当てられる
- [ ] 各グループに英語・日本語対応者が含まれる
- [ ] 同性ペア制限が守られる
- [ ] 同月内に同じメンバーが2回以上割り当てられない（メンバー数が十分な場合）
- [ ] 夫婦が同じグループに割り当てられない（可能な場合）
- [ ] 担当回数が均等になる
- [ ] 最低2週間の間隔が空く
- [ ] 制約を緩和した場合に警告が表示される
- [ ] 手動で割り当てを調整できる
- [ ] 欠席者を差し替えた場合、欠席者の担当回数にカウントされない
