# 仕様書: 割り当て手動調整機能

## 機能概要
自動生成された割り当て結果を、UI上で手動で変更できる機能。特定メンバーの差し替え・欠席対応を行う。

## 背景
既存の `assignment-algorithm.md` では手動調整のユースケース（UC-2, UC-4）を定義済みだが、UI上の操作フローが未定義だった。本仕様書はUI操作とAPIの連携を定義する。

## ユースケース

### UC-1: メンバー差し替え
- **入力**: 割り当てID、差し替え対象メンバー、代わりのメンバー
- **UI操作**:
  1. 割り当て結果画面で、各メンバー名の横に「差し替え」ボタンを表示
  2. ボタンをクリックすると、代わりのメンバーを選択するドロップダウンが表示される
  3. ドロップダウンには有効メンバーのうち、その日に割り当てられていないメンバーを表示
  4. 選択して確定すると、差し替えが実行される
- **正常系**: メンバーが差し替わり、画面が更新される
- **制約チェック**: 差し替え後に制約違反がある場合は警告を表示するが、確定は可能

### UC-2: 欠席対応
- **入力**: 割り当てID、欠席メンバーID、代理メンバーID
- **正常系**: メンバーが差し替わる
- **注意**: 欠席者の年間担当回数にはカウントしない（APIで対応済み）

## 画面仕様

### 割り当て結果の表示
```
4/5（日）
  グループ 1: 田中さん [差し替え] ・ 鈴木さん [差し替え]
  グループ 2: 佐藤さん [差し替え] ・ 山田さん [差し替え]
```

### 差し替えダイアログ
- 対象メンバー名を表示
- 代わりのメンバーを選択するドロップダウン
  - 表示条件: 有効メンバー、かつ当該日に未割り当て
  - 参加可能日の制約に合致するメンバーのみ表示
- 「確定」「キャンセル」ボタン

## API

### PUT /api/assignments/:id/adjust
- **リクエスト**: `{ oldMemberId: string, newMemberId: string }`
- **レスポンス**: 更新後のAssignmentDto
- **エラー**: 割り当てが見つからない、メンバーが見つからない

## 受け入れ基準
- [ ] 各メンバーの横に「差し替え」ボタンが表示される
- [ ] ボタンをクリックすると代わりのメンバーを選択できる
- [ ] 差し替え後に画面が更新される
- [ ] その日に既に割り当てられているメンバーは候補に表示されない
- [ ] 差し替え後に制約違反がある場合は警告が表示される
