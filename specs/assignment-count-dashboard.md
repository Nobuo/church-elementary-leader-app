# 仕様書: 担当回数ダッシュボード

## 機能概要
年間で各メンバーが何回担当したかを一覧表示し、担当回数の偏りを早期に発見するための機能。スケジュール生成時にも年間データを参照して過剰担当を警告する。

## ユースケース

### UC-1: 担当回数一覧表示
- **入力**: 年度
- **出力**: 年度内の全有効メンバーの担当回数一覧
- **表示項目**:
  | 列 | 説明 |
  |----|------|
  | メンバー名 | 氏名 |
  | 担当回数 | 年度内の担当回数 |
  | バー表示 | 回数の視覚化（横棒グラフ） |
- **サマリー**: 表の上部に以下を表示
  - 最大担当回数（とそのメンバー名）
  - 最小担当回数（とそのメンバー名）
  - 平均担当回数
  - 差分（最大 - 最小）
- **並び順**: 担当回数の降順（多い順）
- **色分け**: 平均から大きく外れている場合は赤（多すぎ）/青（少なすぎ）でハイライト

### UC-2: 割り当て生成時の過剰担当警告
- **タイミング**: 自動割り当て生成（UC-1 in assignment-algorithm.md）の実行時
- **判定ロジック**:
  1. 年度内の全有効日曜日の数から、メンバー1人あたりの期待担当回数を算出
     - 期待回数 = (有効日曜日数 × 4) / 有効メンバー数
  2. あるメンバーの担当回数が期待回数の1.5倍を超えている場合、警告を発する
  3. あるメンバーの担当回数が期待回数の0.5倍未満の場合も警告（少なすぎ）
- **出力**: ConstraintViolation として警告を返す（既存の警告表示の仕組みに統合）
- **注意**: 警告は表示のみ。生成自体はブロックしない。

### UC-3: 月間割り当て結果への回数表示
- **場所**: 割り当て結果画面の各メンバー名の横
- **表示**: メンバー名の後に「(3回)」のように年度内累計担当回数を表示
- **例**: `田中さん(3) ・ 鈴木さん(2)`

## API

### GET /api/assignments/counts
- **クエリ**: `fiscalYear=2026`
- **レスポンス**:
```json
{
  "fiscalYear": 2026,
  "summary": {
    "max": { "count": 5, "memberName": "田中太郎" },
    "min": { "count": 2, "memberName": "鈴木二郎" },
    "average": 3.5
  },
  "members": [
    { "id": "...", "name": "田中太郎", "count": 5 },
    { "id": "...", "name": "鈴木二郎", "count": 2 }
  ]
}
```


## ビジネスルール
- 担当回数は年度（4月〜翌3月）でカウント
- 無効化されたメンバーも年度内に担当した分はカウントに含める
- 欠席で差し替えられた場合、欠席者の回数にはカウントしない（既存仕様通り）
- 途中参加メンバーは参加時点からのカウント

## 受け入れ基準
- [ ] 年度内の全メンバーの担当回数が一覧で見える
- [ ] 最大・最小・平均回数がサマリーとして表示される
- [ ] 担当回数が多すぎ/少なすぎるメンバーがハイライトされる
- [ ] 割り当て生成時に過剰担当の警告が表示される
- [ ] 割り当て結果の各メンバー名の横に累計回数が表示される
