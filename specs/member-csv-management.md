# 仕様書: メンバーCSV一括管理 + UIバグ修正

## 機能概要
メンバー情報をCSVで一括ダウンロード・アップロードする機能。また、メンバー追加フォームのUI不具合修正を含む。

## ユースケース

### UC-1: メンバーCSVダウンロード
- **入力**: なし（全メンバー対象）
- **出力**: CSV形式のファイル
- **CSV列**: 氏名, 性別, 言語, 担当区分, メンバー種別, 同性ペア制限, 配偶者名, 参加可能日, 有効/無効
- **文字コード**: UTF-8（BOM付き、Excel対応）
- **注意**: 配偶者はIDではなく氏名で出力する（人間が読めるように）

### UC-2: メンバーCSVアップロード（一括登録・更新）
- **入力**: CSVファイル
- **正常系**:
  1. CSVをパースし、各行のバリデーションを実行
  2. 氏名が既存メンバーと一致する場合は更新、一致しない場合は新規登録
  3. 配偶者は氏名で指定し、CSVの全行処理後に紐付け
  4. 処理結果（成功N件、エラーN件）を返す
- **異常系**:
  - CSVフォーマット不正 → エラー
  - 必須項目が未入力 → 行ごとにエラーを返す（他の行は処理継続）
  - 配偶者名が見つからない → 警告（紐付けスキップ）
- **注意**: アップロードは既存データを全置換ではなく、差分更新（upsert）

### UC-3: UIバグ修正 — 配偶者ドロップダウンのフィルタリング
- **現状の問題**: メンバー種別が「片方の親のみ」「ヘルパー」の場合でも、配偶者選択ドロップダウンが非表示ではあるが候補に残っている
- **修正内容**: 配偶者ドロップダウンの候補に「PARENT_COUPLE」のメンバーまたはまだ配偶者が紐付いていないメンバーのみ表示する
- **追加修正**: 「PARENT_SINGLE」「HELPER」を選択時は配偶者フィールドを完全に非表示にし、値もクリアする

## ドメインモデル

### CSVフォーマット
| 列 | 日本語ヘッダー | 英語ヘッダー | 値の例 |
|----|---------------|-------------|--------|
| 1 | 氏名 | Name | 田中太郎 |
| 2 | 性別 | Gender | MALE / FEMALE |
| 3 | 言語 | Language | JAPANESE / ENGLISH / BOTH |
| 4 | 担当区分 | Grade Group | LOWER / UPPER |
| 5 | メンバー種別 | Member Type | PARENT_COUPLE / PARENT_SINGLE / HELPER |
| 6 | 同性ペア制限 | Same-gender Only | TRUE / FALSE |
| 7 | 配偶者 | Spouse | 鈴木花子（氏名で指定） |
| 8 | 参加可能日 | Available Dates | 2026-04-05;2026-04-12（セミコロン区切り、空欄=全日可） |
| 9 | 有効 | Active | TRUE / FALSE |

## API

### GET /api/members/export/csv
- **クエリ**: `lang=ja|en`
- **レスポンス**: CSVファイル（Content-Disposition: attachment）

### POST /api/members/import/csv
- **リクエスト**: multipart/form-data（CSVファイル）
- **レスポンス**: `{ created: number, updated: number, errors: { row: number, message: string }[] }`

## 受け入れ基準
- [ ] メンバー一覧をCSVでダウンロードできる
- [ ] CSVがExcelで文字化けしない（UTF-8 BOM付き）
- [ ] CSVアップロードで一括登録・更新ができる
- [ ] 既存メンバーは氏名で照合して更新される
- [ ] 配偶者は氏名で指定でき、正しく紐付けられる
- [ ] エラー行があっても他の行は処理される
- [ ] 処理結果が画面に表示される
- [ ] PARENT_SINGLE/HELPER選択時、配偶者ドロップダウンが表示されない
