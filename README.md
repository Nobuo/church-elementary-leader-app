# mach-leader

教会の小学校クラスのリーダー担当を公平に自動割り当てするWebアプリ。

毎週日曜日のクラスで、高学年（4〜6年生）担当2名＋低学年（1〜3年生）担当2名＝計4名のリーダーを、制約条件を考慮しながら自動で組み合わせます。

## 主な機能

- **メンバー管理** — 氏名・性別・言語・担当区分・配偶者関係などを登録
- **スケジュール管理** — 月ごとの日曜日を自動生成、除外日・イベント日の設定
- **自動割り当て** — 7つの制約条件を考慮したGreedy+スコアリングアルゴリズム
- **担当回数ダッシュボード** — 年度内の担当回数を一覧表示、偏りを可視化
- **差し替え** — 割り当て済みメンバーの交代（候補者の推薦付き）
- **エクスポート** — CSV出力（Excel対応UTF-8 BOM付き）、LINE用まとめ文章生成
- **CSV一括管理** — メンバーのCSVダウンロード・アップロードによる一括登録・更新
- **多言語対応** — 日本語・英語の切り替え

## ダウンロード（バイナリ版）

開発環境がなくても、ビルド済みバイナリをダウンロードするだけで使えます。

[最新リリースページ](../../releases/latest) から、お使いの環境に合ったファイルをダウンロードしてください。

| ファイル名 | 対象環境 |
|-----------|---------|
| `leader-app-vX.X.X-darwin-arm64.tar.gz` | **macOS Apple Silicon**（M1 / M2 / M3 / M4 チップ搭載Mac） |
| `leader-app-vX.X.X-darwin-x64.tar.gz` | **macOS Intel**（2020年以前のMac） |
| `leader-app-vX.X.X-windows-x64.zip` | **Windows**（64bit） |

> **どれを選べばいいかわからない場合:**
> - Mac →  メニュー → 「このMacについて」でチップを確認。「Apple M○」なら `darwin-arm64`、「Intel」なら `darwin-x64`
> - Windows → `windows-x64` を選んでください

### macOS での実行

```bash
# 1. ダウンロードしたファイルを展開
tar xzf leader-app-vX.X.X-darwin-arm64.tar.gz

# 2. 検疫属性を解除（重要！）
xattr -cr leader-app

# 3. 起動
cd leader-app
./leader-app
```

ブラウザで **http://localhost:3000** を開きます。

> **「壊れているため開けません」と表示される場合:** macOSはインターネットからダウンロードした署名なしバイナリに検疫属性を付けます。上記の `xattr -cr leader-app` コマンドで解除してください。展開後・起動前に実行するのがポイントです。

### Windows での実行

```
1. ダウンロードしたZIPファイルを展開
2. leader-app フォルダ内の leader-app.exe をダブルクリック
3. ブラウザで http://localhost:3000 を開く
```

> **SmartScreen の警告:** 「WindowsによってPCが保護されました」と表示された場合は、「詳細情報」→「実行」をクリックしてください。

## 開発者向けセットアップ

以下は、ソースコードから開発・実行する場合の手順です。

## 前提条件

- **Node.js** 20 以上（推奨: 22.x）
- **npm**

```bash
# Node.js のバージョン確認
node -v
```

> asdf を使っている場合は `.tool-versions` に `nodejs 22.13.0` が指定されています。

## セットアップ

```bash
git clone <リポジトリURL>
cd mach-leader
npm install
```

これだけで準備完了です。データベース（SQLite）はサーバー初回起動時に自動で作成されます。

## 起動方法

```bash
npm run dev
```

ブラウザで **http://localhost:3000** を開きます。

## 使い方ガイド

アプリには 3 つの画面（タブ）があります。左から順に操作していくのが基本の流れです。

### Step 1: メンバーを登録する

「**メンバー管理**」タブで、リーダー候補者を登録します。

1. 「**追加**」ボタンをクリック
2. 以下の情報を入力:

| 項目 | 説明 |
|------|------|
| 氏名 | 名前（日本語・英語どちらでも可） |
| 性別 | 男性 / 女性 |
| 言語 | 日本語 / 英語 / 両方 — グループ内の言語バランスに影響します |
| 担当区分 | 高学年（4〜6年生） / 低学年（1〜3年生） |
| メンバー種別 | 下記参照 |
| 同性ペア制限 | チェックすると、同性の相手とだけペアを組みます |
| 配偶者 | 夫婦で参加の場合、配偶者を選択（同グループ回避に使用） |

3. 「**保存**」で登録完了

**メンバー種別について:**

| 種別 | 説明 |
|------|------|
| 片方の親のみ | 通常のリーダー（最も一般的） |
| 夫婦で参加 | 配偶者も登録されている場合。同じグループに配置されにくくなります |
| ヘルパー | 通常の日曜日のみ参加可能。イベント日には割り当てられません |

> **目安:** 割り当てが適切に機能するには、高学年・低学年それぞれ **5名以上** の登録を推奨します。

> **CSV一括登録:** 大量のメンバーを登録する場合は「CSVアップロード」が便利です。まず「CSVダウンロード」で書式を確認してから、同じ形式のCSVを作成してアップロードしてください。

### Step 2: スケジュールを設定する

「**スケジュール設定**」タブで、対象月の日曜日を生成します。

1. 画面上部で **年度** と **月** を選択
2. 「**スケジュール生成**」をクリック → その月の全日曜日が表示されます
3. 必要に応じて日曜日ごとに設定:

| ボタン | 動作 |
|--------|------|
| 除外する / 含める | その日を割り当て対象から外す（例: 第5日曜、祝日など） |
| イベント | イベント日に設定する（ヘルパーは割り当て対象外になる） |

### Step 3: 割り当てを自動生成する

「**割り当て結果**」タブで、メンバーの組み合わせを生成します。

1. 年度・月が正しいことを確認
2. 「**自動生成**」をクリック
3. 各日曜日に グループ1（高学年+低学年）・グループ2（高学年+低学年）が表示されます

制約条件に完全に沿えない場合は、画面上部に **警告** が表示されます。該当メンバー名がハイライトされるので、必要に応じて手動で差し替えてください。

### Step 4: 必要に応じて差し替える

自動生成の結果を手動で調整できます。

1. 変更したいメンバー横の「**差し替え**」をクリック
2. 候補者のドロップダウンが表示されます:
   - **★** マーク: 推薦候補（制約違反なし）
   - **⚠** マーク: 制約に注意が必要な候補
   - **(N回)**: その候補の年度内担当回数
3. 候補を選んで「**確定**」

> 過去の日付の割り当てを変更する際は確認ダイアログが表示されます。

### Step 5: エクスポートする

割り当て結果を外部に出力できます。

| ボタン | 出力形式 |
|--------|----------|
| CSV出力 | Excelで開けるCSVファイル（UTF-8 BOM付き）をダウンロード |
| LINE用テキスト | LINEグループに貼り付けられる形式のテキストを生成。ダイアログ内で「コピー」をクリック |

### 担当回数ダッシュボード

割り当て結果の画面下部に、年度内の担当回数が棒グラフで表示されます。

- **最大 / 最少 / 平均 / 差分** の統計が一目でわかります
- 担当が偏っているメンバーには「**多すぎ**」「**少なすぎ**」のラベルが付きます
- 差し替え時にも各候補の担当回数が表示されるので、公平な調整ができます

## 制約条件（優先度順）

自動割り当て時に以下の制約が考慮されます。

| 優先度 | 制約 | 緩和 |
|--------|------|------|
| 必須 | 参加可能日の制約 | 不可 |
| 必須 | 言語バランス（各グループに英語・日本語対応者） | 不可 |
| 強 | 同性ペア制限 | 緩めにくい |
| 弱 | 月内重複回避 | 可 |
| 弱 | 均等な担当回数 | 可 |
| 弱 | 夫婦回避（同一グループ） | 可 |
| 弱 | ペア多様性 | 可 |

> 「弱」の制約は、メンバー数が少ない場合に自動的に緩和されます。

## 年度の運用

- **年度は4月始まり**（4月〜翌3月）
- 年度セレクタで前年度・翌年度も選択可能
- 年度が変わると担当回数カウントがリセットされます
- **年度切り替え時にやること:**
  - メンバーの担当区分（高学年 ↔ 低学年）を必要に応じて更新
  - 卒業・引越しなどで参加しなくなったメンバーを「無効化」
  - 新しいメンバーを追加

> 無効化したメンバーは「無効メンバーも表示」チェックで確認できます。再度有効化はできないため、必要な場合は新規登録してください。

## 言語の切り替え

画面左上のセレクタで **日本語 / English** を切り替えられます。CSV出力・LINE用テキストにも言語設定が反映されます。

## トラブルシューティング

### `npm test` で `NODE_MODULE_VERSION` エラーが出る場合

```
Error: The module '.../better_sqlite3.node' was compiled against a different Node.js version
using NODE_MODULE_VERSION XXX. This version of Node.js requires NODE_MODULE_VERSION YYY.
```

`better-sqlite3` はネイティブアドオン（C++）のため、`npm install` 時のNode.jsバージョン向けにコンパイルされます。Node.jsのバージョンが変わった場合（asdfでの切り替え、Bunインストール後など）、以下のコマンドで再コンパイルしてください:

```bash
npm rebuild better-sqlite3
```
