# グループを学年区分ごとにまとめる

## 日付: 2026-03-20

## アイデアの概要

現在の割り当て結果は、各グループに高学年1人＋低学年1人が配置されている。
ユーザーの要望は、**グループ1＝高学年2人、グループ2＝低学年2人**とすること。

### 現状（変更前）

```
4/5（日）
  グループ 1: [高] メンバーA ・ [低] メンバーB
  グループ 2: [高] メンバーC ・ [低] メンバーD
```

### 希望（変更後）

```
4/5（日）
  グループ 1: メンバーA ・ メンバーC    ← 高学年2人
  グループ 2: メンバーB ・ メンバーD    ← 低学年2人
```

## 背景

- 実際の運用では、高学年（4〜6年生）と低学年（1〜3年生）は別の教室で活動する
- グループ＝教室の担当ペアなので、同じ学年区分の2人がペアになるのが自然
- 現状の「高＋低」ペアは、4人で1つのクラスをリードするモデルを想定していたが、実際は2教室に分かれている

## データ構造への影響

### 影響あり

1. **Assignment エンティティの規約変更**
   - 現在: `memberIds[0]` = UPPER枠、`memberIds[1]` = LOWER枠（1グループ内に高低混在）
   - 変更後: `memberIds[0]` と `memberIds[1]` は同じ学年区分の2人
   - `groupNumber=1` → UPPER グループ、`groupNumber=2` → LOWER グループ（新しい規約）

2. **割り当てアルゴリズム（`generateAssignments`）の大幅変更**
   - 現在: `pickBestPair(upperPool, lowerPool)` → 1人ずつ選出
   - 変更後: `pickBestPair(upperPool, upperPool)` と `pickBestPair(lowerPool, lowerPool)` → 同区分から2人選出
   - ペアスコアリングのロジック変更が必要

3. **言語バランス制約の適用範囲変更**
   - 現在: 各ペア（高+低）内で言語バランスを確保
   - 変更後: 各学年グループ（同区分2人）内で言語バランスを確保
   - 分級日のバイリンガル要件は「4人中2人以上」→ 変わらないが、チェック方法が変わる

4. **AssignmentDto の `role` フィールド**
   - 現在: `idx === 0 ? UPPER : LOWER`（位置ベース）
   - 変更後: グループ番号ベース（`groupNumber === 1 ? UPPER : LOWER`）、または不要になる可能性

5. **UI の [高]/[低] ラベル表示**
   - 個別メンバーのラベルは**残す**（メンバーの登録上の区分をそのまま表示）
   - 横断時も `[高]` のまま表示（`[低←高]` のような矢印表記は不要）
   - 低学年グループに `[高]` ラベルのメンバーがいれば、横断が発生していることがわかる
   - グループヘッダーにも「高学年」「低学年」を表示するとさらにわかりやすい

6. **区分横断（crossover）ロジック**
   - 分級日にUPPERのBOTHメンバーがLOWERグループ（グループ2）に入る場合の表示が変わる

7. **差し替え（adjust）ロジック**
   - 候補フィルタリング: グループ番号から学年区分を判定

### 影響なし

- DB スキーマ自体（group_number, member_id_1, member_id_2）は変更不要
- 既存のマイグレーションに影響なし（規約の変更のみ）

## 期待される効果

- 実際の運用（高学年教室・低学年教室の2教室制）と画面表示が一致する
- ユーザーが直感的に「誰がどの教室を担当するか」を把握できる
- [高]/[低] ラベルにより、区分横断時に「本来は別グループの人」が視覚的にわかる
