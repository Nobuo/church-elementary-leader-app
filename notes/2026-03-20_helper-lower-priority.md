# ヘルパーの割り当て優先度を親より下げる

## 日付: 2026-03-20

## 要望

ヘルパー（HELPER）は手伝いの立場なので、スコアが同程度であれば親（PARENT_COUPLE / PARENT_SINGLE）を優先的にアサインしたい。

## 背景

### 現在の仕組み

- `scorePair()` ではメンバータイプによる優先度差はない
- イベント日にはヘルパーが除外されるが、通常日は親と同等に選出される
- 均等配分（count差 × 50）が主な選出基準で、ヘルパーも同じ重みでカウントされる

### 課題

- ヘルパーは「助けてくれる方」であり、親が責任を持つべき活動
- 同スコアのペアが複数ある場合、ヘルパーを含むペアと親のみのペアが等確率で選ばれる
- 親が十分いる場合でも、ヘルパーが不必要に多く選出される可能性がある

## 改善案

### `scorePair()` にヘルパーペナルティを追加

```typescript
// SOFT: HELPER deferral — prefer PARENT members when scores are close
for (const m of [member1, member2]) {
  if (m.memberType === MemberType.HELPER) {
    score += 5; // 小さなペナルティで親を優先
  }
}
```

### ペナルティ値の検討

- **+5（推奨）**: 微小なペナルティで、同スコアなら親を優先するが、均等配分（+50/回）やペア多様性（+10/回）には負けるため、ヘルパーが極端に少なくなることはない
- **+10**: やや強め。ペア多様性と同等の重み
- **+1〜2**: 同スコアのタイブレークにのみ効く

### 注意点

- ペナルティが大きすぎるとヘルパーがほとんど選出されなくなり、均等配分が崩れる
- 均等配分ペナルティ（+50/回差）の方が大きいため、親のcount > ヘルパーのcountの状況ではヘルパーが選ばれる（これは正しい動作）
- ヘルパーの中でもANY学年の方は両プールに入るため、選出頻度が高くなりやすい → ペナルティで自然に抑制される効果もある

## 期待される効果

- 親が同程度の条件であれば優先的にアサインされる
- ヘルパーは「補助」としての立場が反映される
- 均等配分は維持される（count差がペナルティを上回るため）
