# ソフトペナルティによる決定的排除問題

## 日付: 2026-03-21

## 現象

シャッフル（タスク023）導入後も、毎回同じメンバーが0回のまま残る。スクリーンショートでは16人が1回、7人が0回。

## 原因分析

### 数学的制約

4日間 × 2グループ × 2人 = 16スロット。23人中16人しか割り当てられない。**7人は必ず0回になる。**

### 月内重複ペナルティ(+100)の効果

月内重複ペナルティにより、一度割り当てられたメンバーは次の日に選ばれにくい。結果、アルゴリズムは16人のユニークメンバーを選ぶ。これ自体は正しい動作。

### ソフトペナルティの決定的排除

問題は最後の日（4日目）で11人から4人を選ぶとき。全員count=0だが、ソフトペナルティに差がある:

| ペナルティ | 値 | 影響 |
|-----------|-----|------|
| HELPER後回し | +5/人 | HELPERは常にPARENTより不利 |
| BOTH温存 | +3/人 | BOTHは合同日に常に不利 |
| 配偶者回避 | +30/ペア | PARENT_COUPLEの配偶者同士は不利 |

これらは**入力データに依存する固定値**で、シャッフルしても変わらない。タイブレーク（同スコア時のランダム選択）はスコアが等しい時にしか発動しないため、HELPER+5やBOTH+3の差があると常に同じメンバーが負ける。

### 具体例

- 非HELPER・非BOTH ペア: スコア = 0
- HELPER ペア（2人とも）: スコア = 10
- BOTH ペア（合同日、2人とも）: スコア = 6
- HELPER+BOTH ペア: スコア = 8

シャッフルタイブレークはスコア0のペア同士では発動するが、スコア0 vs スコア10では必ずスコア0が勝つ。

## 改善案

### 案1: スコアにランダムジッタを追加

```typescript
// scorePair() の最後に
score += Math.random() * 8; // 0〜8のランダム値
```

HELPER(+5)やBOTH(+3)の差を時々打ち消し、選ばれるチャンスを与える。ただしHELPER後回しの効果が弱まる。

### 案2: 月内「未割り当て」ボーナス

月内でまだ1回も割り当てられていないメンバーにボーナスを付与。

```typescript
// 月内未割り当てボーナス
for (const m of [member1, member2]) {
  const neverAssigned = !monthAssignments.some((a) => a.containsMember(m.id));
  if (neverAssigned) {
    score -= 15; // 未割り当てを優先
  }
}
```

これにより、最後の日に残った未割り当てメンバーがHELPER/BOTHでも優先される。

### 案3: HELPER/BOTH ペナルティの条件付き適用

全員が同じcount時はHELPER/BOTHペナルティを適用しない。

## 試行結果

| # | 施策 | 結果 |
|---|------|------|
| 1 | 月内未割り当てボーナス(-15) | HELPER4人が10回とも0回（効果なし） |
| 2 | 未割り当てボーナス(-30) | 同上（全未割り当てに等しく適用されるため相対差が変わらない） |
| 3 | HELPERペナルティを条件付き + ボーナス(-30) | 固定排除解消、ランダム |
| 4 | HELPER+BOTH条件付き + ボーナス(-30) | 一部の日でグループ形成失敗（BOTHが使い切られる） |
| 5 | **HELPERペナルティ条件付きのみ** | **最良: 固定排除なし、全制約遵守** |

## 採用した修正

**HELPERペナルティを「月内で既に割り当て済みの場合のみ」に変更**

```typescript
// 変更前
if (m.memberType === MemberType.HELPER) {
  score += 5;
}

// 変更後
if (m.memberType === MemberType.HELPER) {
  const alreadyAssigned = monthAssignments.some((a) => a.containsMember(m.id));
  if (alreadyAssigned) {
    score += 5;
  }
}
```

これにより:
- 初回割り当てではHELPERとPARENTが対等に競合
- 2回目以降はHELPERが後回し（従来通り）
- BOTH温存は常時適用（変更すると言語バランスが崩れるため）
