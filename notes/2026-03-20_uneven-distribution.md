# 割り当て回数の偏り（6ヶ月間）

## 日付: 2026-03-20

## 問題の概要

2026年4月〜9月（26日間）の自動生成で、割り当て回数に大きな偏りが発生。

| 回数 | メンバー数 | 特徴 |
|------|-----------|------|
| 6回 | 7人 | LOWER全般 + UPPER BOTH |
| 5回 | 7人 | LOWER JP + UPPER BOTH |
| 3〜4回 | 8人 | UPPER JP |
| 1回 | 1人 | 日付制限あり |

## 原因分析

### 原因1: グループ人数の不均衡（構造的）

| グループ | 人数 | 26日間のスロット | 1人あたり平均 |
|----------|------|-----------------|-------------|
| UPPER (G1) | 13人 | 52スロット | **4.0回** |
| LOWER (G2) | 10人 | 52スロット | **5.2回** |

LOWERは人数が少ないため、1人あたりの担当回数が構造的に多くなる。

### 原因2: 言語制約によるBOTH強制（構造的）

UPPERにEN専門が0人のため、**毎回最低1人のBOTHが必須**。

- UPPER BOTH 5人 → 26日 × 1 BOTH = 26スロット / 5人 = **5.2回**
- UPPER JP 8人 → 残り26スロット / 8人 = **3.25回**

BOTHメンバーはJPメンバーの約1.6倍の頻度で割り当てられる。これは制約上避けられない。

### 原因3: 同スコア時の配列順バイアス（アルゴリズム的 — 修正可能）

`pickBestPairSameGrade()` でスコアが同点の場合、**最初に見つかったペアが常に勝つ**:

```typescript
for (let i = 0; i < candidates.length; i++) {
  for (let j = i + 1; j < candidates.length; j++) {
    if (score < bestScore) {  // 同点は更新されない
      bestPair = ...;
    }
  }
}
```

メンバー配列の順序はDBの取得順に依存し固定。配列の先頭にいるメンバーほどタイブレークで有利になり、6ヶ月間で累積する。

## 改善案

### 案A: 候補配列のシャッフル（推奨）

`pickBestPairSameGrade()` の呼び出し前に候補をシャッフルする。

```typescript
const shuffled = [...candidates].sort(() => Math.random() - 0.5);
```

- **効果**: 同スコア時のバイアスを解消。構造的な偏りは解消しないが、特定の人が常に有利/不利になることを防ぐ
- **影響**: テストの決定性が失われる（シード付きランダムで対応可能）

### 案B: 同スコア時のランダム選択

スコアが同点の場合、一定確率で入れ替える。

```typescript
if (score < bestScore || (score === bestScore && Math.random() < 0.5)) {
```

- **効果**: 案Aと同等
- **影響**: より局所的な変更

### 案C: グループ人数の考慮（将来検討）

均等配分のペナルティ計算で、グループ内の人数差を考慮した正規化を行う。LOWERの回数が多いのは構造的に避けられないが、「UPPER JPは3回なのにLOWER ENは6回」という差を緩和できる可能性がある。

## 期待される効果

| 改善案 | 最大偏差(現状6-3=3) | 備考 |
|--------|-------------------|------|
| 案A/B | 2程度に縮小 | 構造差は残る |
| 案C | 1程度に縮小 | 設計変更が必要 |
