# Spec: 仕様書作成

要件やアイデアから詳細な仕様書を作成し、`specs/` 配下に保存してください。

## 手順
1. `requirements.md` と関連する `notes/` を読み込む
2. `.claude/architecture.md` の設計方針を確認する
3. 機能単位で仕様書を作成する
4. `specs/` に保存する

## 仕様書に含める内容
- 機能概要
- ユースケース（正常系・異常系）
- 入出力の定義
- ドメインモデル（DDD観点）
- 制約・ビジネスルール
- 受け入れ基準（テスト観点）

## 出力形式
- ファイル名: `specs/<feature-name>.md`
