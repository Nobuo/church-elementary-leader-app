# Review: コードレビュー

実装されたコードを仕様書・設計方針に基づいてレビューしてください。

## 手順
1. 対象の変更差分を確認する（`git diff` または指定ファイル）
2. 関連する仕様書 (`specs/`) を確認する
3. `.claude/architecture.md` と `.claude/coding-rules.md` に照らしてレビューする
4. レビュー結果を出力する

## レビュー観点
- 仕様との整合性
- DDD・クリーンアーキテクチャへの準拠
- コーディング規約の遵守
- テストの十分性
- エッジケースの考慮
- セキュリティ上の懸念
- パフォーマンスへの影響

## 出力形式
- 問題点があれば具体的なファイル名・行番号とともに指摘
- 改善提案を含める
- 重大度: Critical / Warning / Info で分類
