# プロジェクトコンテキスト

## プロジェクト名
小学校クラスリーダー組み合わせアプリ (mach-leader)

## 概要
教会の小学校クラスのリーダーを、親たちで公平に組み合わせるためのアプリケーション。

## 開発アプローチ
- Spec駆動開発: アイデア → 仕様 → 計画 → 実装 → レビュー
- DDD（ドメイン駆動設計）
- クリーンアーキテクチャ
- テストファースト

## 技術スタック
- 言語: TypeScript
- 永続層: SQLite（取り替え可能な設計）
- プラットフォーム: クロスプラットフォーム（Mac / Windows）
- 実行環境: ローカル

## 開発フロー
1. `/idea` - アイデア・要望を整理して `notes/` に保存
2. `/spec` - 仕様書を作成して `specs/` に保存
3. `/plan` - 実装計画を立てて `tasks/` に保存
4. `/implement` - タスクに基づいて実装
5. `/review` - コードレビュー
